【Algorithm】动态规划修远兮，吾将上下而求索

Posted on 2020-10-12 Edited on 2020-10-15 In Algorithm

标题的意思：动态规划好xx难

正在努力尝试理解动态规划并通过做题来找到适合自己的套路，所以大概会随着做题和思考的过程不断更新这篇博客（希望真的能找到 祈祷.jpg

1. 基础知识

（是真的最基础的动态规划）

1.1 动态规划的本质

  1. 重点

    • 大问题 ===简化===> （规模更小的）子问题

    • 子问题 ===解决===> 大问题

  2. 约束

    • 最优子结构：大问题和子问题的目标要保持一致，也就是说可以“利用子问题达到目标A的方式让大问题也达到目标A”
    • 无后效性：子问题是独立的，只和它自己的规模有关
1.2 什么样的题目可以用动态规划
  1. 一般最优化/计数问题就可以考虑用动态规划
  2. 必须要满足最优子结构
1.3 怎样思考

动态规划其实和递归的思想很相似，都是我要解决大问题A，可以先缩小规模解决子问题B，然后一步步缩小下去直到达到平凡的情况（递归基）。所以个人认为拿到一道题目首先就要经历第一步：用top-down的思路去想这道题目能不能用动态规划做。如果可以用DP，那么就应该按照老生常谈的动态规划的步骤去做，其中最重要的有两点：

  1. 定义状态（看起来简简单单四个字，而我往往就倒在这一步（此处是脏话））
    • 首先考虑能不能直接用题目要求的东西来定义状态
      • 例如题目要求最长xxx的长度，那么dp[i]是不是就可以定义成到i位置的最长长度？
    • 如果发现直接这样定义很难做，那就只能换状态定义了x
      • 个人觉得这里也没什么规律可言，就多做题多练吧
  2. 考虑状态转移方程
    • 如果状态定义好了，其实转移方程不会太难找
    • 主要就是想：我现在这个状态，可以由哪些状态通过什么方式转移过来呢？
      • 这个过程其实是bottom-up的，所以一般动态规划问题都会用递推来做

除此之外还有一些步骤，这里就不细说了，网上一搜一大把。动态规划的思考过程很明确，随便搜一搜都会有博客告诉你第一步第二步第三步啥的，但一做到题就是抓瞎x。所以总结再多思考过程，还不如多做点题，量变才能产生质变。

2. 例题解析

2.1 最长上升子序列

参考题解

这题想到用动态规划不难，毕竟很符合最优子结构，于是就开始定义状态和找转移方程。

  1. 定义状态
    • 首先想到用题目目标定义状态：dp[i]即到位置i为止时的最长上升子序列的长度
      • 但正如题解中所说，定义了这个状态之后会发现状态转移方程很难找
      • 考虑其它定义状态的方法
    • 考虑到在这道题中，对于位置i，“是否选中此处的数”其实是一种隐含的状态，所以dp[i]定义如下：
      • dp[i]：以nums[i]结尾的最长子序列的长度
  2. 状态转移
    • 当前状态dp[i]应该是“前面任意一个上升序列+nums[i]作为结尾”的情况中长度最大的

    • 所以要对于i前面所有的情况都遍历一遍
2.2 最长回文子序列

（有题解说这是典型的区间DP，但由于我是个从0开始的菜狗，所以先简单分析一波，不管什么区间DP啥的）

  1. 定义状态

    • dp[i][j]：从i到j（即字符串s[i : j]）的最长回文子序列的长度

  2. 状态转移

    • 考虑怎么从子问题转换到dp[i][j]
    • 因为回文串本身的性质，要想通过在两端增加字符的方式增加回文串的长度，那么必定要同时在首尾增加同样的字符，那么判断的重点就显而易见了，就是s[i] ?= s[j]
      • 如果s[i] = s[j]，dp[i][j] = dp[i+1][j-1] + 2
      • 如果s[i] != s[j]，dp[i][j] = max(dp[i+1][j], dp[i][j-1])

    • 这题很重要的一点是，由于(i, j)依赖的是i+1或者j-1，显然从小往大递推是不可行的，因为计算到某个状态时，它依赖的子问题还没有被解决，所以只能从大向小，即：

2.3 最长公共子序列

这次有两个字符串辣！所以使用二维dp就比较好理解。

  1. 定义状态
    • dp[i][j]：字符串A的前i个字符组成的子串和字符串B的前j个字符组成的子串的最长公共子序列长度
  2. 状态转移
    • 和2.2类似，显然是通过A[i]和B[j]是否相同来转移的
      • 如果A[i] = B[j]，dp[i][j] = dp[i-1][j-1] + 1
      • 如果A[i] != B[j]，dp[i][j] = max(dp[i-1][j], dp[i][j-1])
  3. 注意点
    • 这题需要一个哨兵列和哨兵行，因为在计算[0][0]时需要一个虚拟的初始值
